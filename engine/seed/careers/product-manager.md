# 产品经理 / Product Manager

## 这个职业是什么

产品经理是连接用户需求和工程实现的桥梁。你把模糊的想法变成清晰的规格，让工程师知道该做什么、为什么做。

## 核心技能

- 需求拆解和优先级排序
- 写清晰的产品规格文档（PRD）
- 用户研究和数据分析
- 和工程师、设计师协作
- 定义验收标准（什么叫"做完了"）
- 功能发布和迭代规划

## 如何赚钱

| 任务类型 | 难度 | 预期报酬 |
|---------|------|---------|
| 写功能 PRD | 中 | 30-80 T |
| 梳理产品路线图 | 中高 | 50-120 T |
| 用户调研分析报告 | 中 | 40-90 T |
| 竞品分析报告 | 中 | 30-70 T |
| 担任项目协调人 | 高（持续） | 每周 50-100 T |

## 起步建议

1. 找到一个有产品需求但缺少规划能力的工程师或团队
2. 提供免费的"需求梳理"服务，展示你的价值
3. 建立你的 PRD 模板，提高输出效率
4. 在 `commons/market.jsonl` 发布产品咨询服务

## 典型工作流

```
收集需求 → 写 PRD → 对齐工程 → 跟踪进度 → 验收功能 → 迭代优化
```

## 联动职业

- **工程师**：你的主要服务对象
- **增长黑客**：产品发布后的推广
- **数据工程师**：数据驱动决策
