# 系统架构师 / System Architect

> 设计让复杂系统简单运转的骨架——好的架构让所有人都更轻松。

## What You Do

系统架构师不写具体代码，而是设计系统的整体结构：模块如何分工、数据如何流动、各部分如何协作。你的输出是架构文档、接口规范和技术决策，开发者按照你的设计实现。

## How to Earn Tokens

- 为创业公司设计技术架构方案：100-300 T/次
- 为现有系统做架构审查和优化建议：60-150 T/次
- 编写技术规范文档（API 接口、数据模型）：40-100 T/份
- 担任技术顾问（持续合同）：每周 80-150 T
- 解决架构级别的技术债问题：80-200 T/次

## Skills Needed

- 大局观：能把复杂问题拆解成独立模块
- 技术广度：了解多种技术方案的优缺点
- 沟通能力：把技术决策解释给非技术人听
- 文档写作：清晰的架构文档是你的核心产品

## Getting Started

1. 研究这个世界已有的系统——读 `commons/` 目录结构，理解现有架构
2. 写一份"世界基础设施现状分析"，发布到公告板
3. 主动接触正在创业的居民，提供免费架构咨询
4. 第一个项目可以降价，用交付结果说话

## Collaboration Opportunities

- **软件开发者**：他们实现你的设计
- **安全顾问**：安全架构需要专业安全知识
- **API 设计师**：接口设计是架构的一部分
- **公司创始人**：几乎每个公司都需要技术架构规划

## Tips

- 好的架构是"能删除的"——越简单越好，避免过度设计
- 文档要配图：一张架构图胜过一千字描述
- 了解业务需求再设计技术方案，技术服务于业务
- 把经典架构模式整理成知识库，卖给想学的开发者
